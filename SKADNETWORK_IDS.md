## Why This Matters

**iOS 14+ Requirement:**
- Apple's App Tracking Transparency (ATT) framework requires SKAdNetwork IDs
- Without these, your ad revenue will be significantly reduced
- Ad networks use these IDs for attribution without violating privacy

**Impact:**
- ✅ With SKAdNetwork IDs: Full ad revenue potential
- ❌ Without: 30-50% revenue loss on iOS 14+

## How to Add to Your Project

### Option 1: Edit in Xcode (Recommended)
1. Open `ios/Runner.xcworkspace`
2. Select `Info.plist` in Project Navigator
3. Right-click in editor → Open As → Source Code
4. Find the closing `</dict>` tag near the end
5. Paste the entire SKAdNetworkItems array BEFORE `</dict>`
6. Save file (⌘S)

### Option 2: Edit in Text Editor
1. Open `ios/Runner/Info.plist` in any text editor
2. Find the closing `</dict>` tag (second to last line)
3. Paste the SKAdNetworkItems array BEFORE `</dict>`
4. Save file

## Verification

After adding, your Info.plist structure should look like:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <!-- All your existing keys -->
    <key>CFBundleIdentifier</key>
    <string>...</string>
    
    <!-- ... more keys ... -->
    
    <!-- GADApplicationIdentifier MUST be here -->
    <key>GADApplicationIdentifier</key>
    <string>ca-app-pub-XXXXX~YYYYY</string>
    
    <!-- SKAdNetwork IDs -->
    <key>SKAdNetworkItems</key>
    <array>
        <!-- All the items from above -->
    </array>
    
    <!-- ... more keys ... -->
    
</dict>
</plist>
```


2. Archive and upload to TestFlight

3. Test ads in TestFlight build (not simulator)

4. Verify ads show properly and attribution works

## Common Mistakes

❌ **Wrong location:** SKAdNetworkItems must be at root level of plist dict
❌ **Missing GADApplicationIdentifier:** Both are required
❌ **Typos in identifiers:** Copy-paste exactly as shown
❌ **Old list:** Use current list from Google docs
❌ **Missing closing tags:** Ensure valid XML structure

## Validation Command

Check if your Info.plist is valid XML:

```bash
plutil -lint ios/Runner/Info.plist
```

Should output: `ios/Runner/Info.plist: OK`
